# AWS SAA 필기들

- 공부하는 이유: AWS의 클라우드 생태계 적응과 사용법, 클라우드 지식 체득.

## Lecture 1.
- 강의에 대한 개요와 aws계정 생성. 클라우드 지식뿐 아니라 실제 aws사용법까지 알려줄듯.

## Lecture 3.
- aws의 리전, AZ에 대한 설명.
  - 리전은 데이터센터 클러스터링 된 지리적 영역이다. 잘 골라서 지연시간을 최소화해보자.
  - AZ(가용성 구역)는 데이터센터의 집합이고, 보통 3개정도 존재하며 실질적인 작업을 한다.
  - 엣지 로케이션은 클라이언트에게 가벼운 데이터를 빠르게 전달하기위한 캐시 서버다.

## Lecture 4.
- IAM, AWS CLI
  - IAM은 AWS환경에서의 개발에 핵심적인 기능이다.
    - IAM user를 생성해 그룹화 및 권한 부여와 이를 통한 서비스 대 서비스 통신을 가능하게한다.
    - IAM role을 사용해 서비스가 통신에 쓰이는 api, 키값 등에 임시 접근 가능한 권한을 발급해줄 수 있다.
    - 보통 개발시 루트 계정은 사용하지않고, 보안을 위해 그와 유사한 권한을 가진 IAM 계정으로 개발한다.
    - IAM 정책 구조는 json파일로 되어있고, sid, effect(접근 허용여부), principal(정책이 적용되는 대상(명확한 대상에 적용되는 경우 생략)), action(정책 대상자가 할 수 있는 API 함수들), resource(정책 대상이 작업에 쓸 수 있는 리소스들) condition(정책 적용 전 추가적인 제약 조건)이 포함된다.
    - MFA는 IAM 계정에 사용하는 암호+물리적 암호장치를 말한다.
      - 각각의 토큰을 만드는 가상 장치와 하나의 키값을 가지는 물리적 장치 두 방식이 존재한다. 둘 다 하나의 장치로 여러 사용자를 담당할 수 있다.
      - <details>
        <summary>보안에 대한 추가 지식</summary>
        정보보안 분야에서 인증은 3가지 요소로 나뉜다.
        1. 지식 기반: 머릿속으로 기억하고있는 것. 비밀번호 등.
        2. 소유 기반: 물리적으로 가지고있는 것. 스마트폰, 하드웨어 보안키 등.
        3. 생체 기반: 나의 고유한 신체적 특징. 지문, 얼굴 등.
        MFA는 이 3개중 최소 2개 이상을 조합하여 신원을 증명하는 방식이다.
