# AWS SAA 필기들

- 공부하는 이유: AWS의 클라우드 생태계 적응과 사용법, 클라우드 지식 체득.

## Lecture 1.
- 강의에 대한 개요와 aws계정 생성. 클라우드 지식뿐 아니라 실제 aws사용법까지 알려줄듯.

## Lecture 3.
- aws의 리전, AZ에 대한 설명.
  - 리전은 데이터센터 클러스터링 된 지리적 영역이다. 잘 골라서 지연시간을 최소화해보자.
  - AZ(가용성 구역)는 데이터센터의 집합이고, 보통 3개정도 존재하며 실질적인 작업을 한다.
  - 엣지 로케이션은 클라이언트에게 가벼운 데이터를 빠르게 전달하기위한 캐시 서버다.

## Lecture 4.
- AWS 서비스
  - AWS에서 전세계 데이터센터에 미리 빌드하여 개발자에게 API형태로 제공하는 하나의 제품이다.
  - 모든 기능을 하나의 제품에 넣으면 비효율적이고, 사용자도 가격이 부담스러울것.
  - EC2, S3, IAM및 그 외 다양한 서비스가 존재한다.
- IAM, AWS CLI
  - IAM은 AWS환경에서 권한과 보안을 통제하는, 개발에 핵심적인 기능이다.
    - IAM user를 생성해 그룹화 및 권한 부여를 통해 사람 및 외부 시스템의 접근을 통제하고, 서비스 대 서비스 통신은 IAM Role을 통해 가능하게 한다.
      - user와 role의 용법상 차이로는 전자는 aws에 상시 접근해야 하는경우, 후자는 그정돈 아니라 임시 토큰만으로 해결 될 경우에 사용한다.
    - 보통 개발시 루트 계정은 사용하지않고, 보안을 위해 그와 유사한 권한을 가진 IAM 계정으로 개발한다.
    - IAM 정책 구조는 json파일로 되어있고, version(문법 버전. 보통 2012-10-17로 고정), sid, effect(접근 허용여부), principal(정책이 적용되는 대상(명확한 대상에 적용되는 경우 생략)), action(정책 대상자가 할 수 있는 API 함수들), resource(정책 대상이 작업에 쓸 수 있는 리소스들) condition(정책 적용 전 추가적인 제약 조건)이 포함된다.
    - MFA는 IAM 계정에 사용하는 암호+물리적 암호장치를 말한다.
      - 각각의 토큰을 만드는 가상 장치와 하나의 키값을 가지는 물리적 장치 두 방식이 존재한다. 둘 다 하나의 장치로 여러 사용자를 담당할 수 있다. 물론 암호는 각각 다른 대칭키를 사용한다.
      - <details>
        <summary>보안에 대한 추가 지식</summary>
        정보보안 분야에서 인증은 3가지 요소로 나뉜다.
     
        
        1. 지식 기반: 머릿속으로 기억하고있는 것. 비밀번호 등.  
        2. 소유 기반: 물리적으로 가지고있는 것. 스마트폰, 하드웨어 보안키 등.  
        3. 생체 기반: 나의 고유한 신체적 특징. 지문, 얼굴 등.
     
       
        MFA는 이 3개중 최소 2개 이상을 조합하여 신원을 증명하는 방식이다.
      </details>
  - AWS CLI는 AWS에 터미널로 접근하여 상호작용하는 방식이다.
    - 과거에는 IAM 사용자의 정적 액세스키를 발급해 썼으나 보안상 권장되지 않으며, 현재는 IAM Identity Center(SSO)를 통한 임시 토큰 로그인 방식을 권장한다.
## Lecture 5.
- 청구 서비스에서 예산 관리와 모니터링을 할 수 있는데 아주 유용하고 필수적이다.
- EC2(Elastic Compute Cloud): aws에서 가상 머신을 대여해서 이것저것에 쓸 수 있는 서비스.
  - 인스턴스 타입: General: 통상 타입, Compute: 컴퓨팅 최적화, Ram: 데이터 처리에 적합, Storage: 저장 및 접근 최적화.
  - 보안 그룹: EC2 인스턴스의 방화벽 역할. 인스턴스 앞에서 트래픽을 먼저 확인해준다. 포트와 승인된 ip주소값, 네트워크 입출력 등을 조절한다. 다른 보안 그룹을 참조해서 승인된 보안 그룹에서 나온 트래픽일 경우 바로 들여보내거나 할 수 있다.
  - 인스턴스 조작: 인스턴스를 조작한다는 것은 특정한 컴퓨터를 외부에서 제어하겠다는 말과 동일하다. 조작하기 위해선 특별한 방법들을 써야한다.
    - SSH(Secure Shell): 22번 포트로, 컴퓨터의 원격 제어에 쓰인다. 키값이 담긴 pem파일을 참조해서 cmd창으로 인스턴스에 접근해 작업한다.
    - PuTTY: 옛날 os에서나 쓰인다. SSH랑 사용방법은 유사한데 좀 더 구식이다.
    - EC2 Instance Connect: aws 콘솔에서 자체적으로 지원하는 접근 기능으로 키가 필요없어 매우 편리하다. SSH를 사용한다.
    - IAM role을 인스턴스에 입혀서 aws관련 상호작용을 할 수 있다.
